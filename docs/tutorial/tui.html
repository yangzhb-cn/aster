Package · com.aster.ui.tui

tui 包：把 Agent 事件变成终端里的结构化消息。

TUI 不直接理解模型协议，也不直接执行工具。它只接收 `AgentEventEnvelope`，取出其中的 `event` 后转换成可展示状态：用户块、thinking 块、工具块、assistant 正文块，以及输入框下方的 token 预算行。

UI 状态按块保存，渲染时再决定颜色、折叠和 Markdown 表现。

流程图

  -> 判断事件类型
  -> thinking/token/tool/usage 分别更新

TUI 不把 usage 和后台通知写入对话历史，避免污染聊天语义。

源码对照：为什么不用字符串 history

UiBlock.java 
public sealed interface UiBlock permits UserBlock, AssistantBlock, ReasoningBlock, ToolBlock, SystemBlock, ErrorBlock {
}
中文解释

旧做法是 `List<String> history`，只能靠前缀判断颜色。

现在每种历史消息都有类型，TUI 可以分别画 thinking、工具和正文。

usage 不属于历史消息，所以它不进 UiBlock，而是更新底部预算行。

这也是后续做折叠、Markdown、状态样式的基础。

源码对照：事件转展示块

TuiAgentEventHandler.java 
public void onEvent(AgentEventEnvelope envelope) {
    AgentEvent event = envelope.event();

    if (event instanceof AgentEvent.AssistantToken token) {
        window.appendAssistantToken(token.text());
        return;
    }
    if (event instanceof AgentEvent.ReasoningToken token) {
        window.appendReasoningToken(token.text());
        return;
    }
    if (event instanceof AgentEvent.UsageReported usage) {
        window.showUsage(usage.usage(), usage.maxContextTokens());
        return;
    }
}
中文解释

正文和 DeepSeek thinking 分成两个事件。

TUI 因此可以把 reasoning_content 显示成灰色 thinking 块。

usage 也单独成事件，避免混进 assistant 正文或历史消息。

TUI 不需要处理所有生命周期事件；Web SSE、日志可以消费同一批 AgentEventEnvelope。

TUI 当前只用 envelope.event()，Web 或日志后续可以使用 envelope.meta() 做顺序、审计和关联。

这比把所有 token 拼进一段 assistant 文本更清楚。

源码对照：输入框下方的 token 预算行

AgentTuiWindow.java 
public void showUsage(TokenUsage usage, int maxContextTokens) {
    enqueue(() -> {
        lastUsage = usage;
        lastMaxContextTokens = maxContextTokens;
        status = "usage input=" + usage.inputTokens() + " output=" + usage.outputTokens();
        dirty = true;
    });
}
中文解释

这里展示的是这一轮 LLM 请求的真实用量，不是本地估算。

input 优先按 cache + miss 计算，output 对应 completion tokens。

cache / miss 对应输入缓存命中和未命中，供应商不返回时就是 0。

context 显示 input / maxContextTokens，用来观察当前上下文窗口占比。

它放在输入框下面，避免把统计信息混进对话历史。

源码对照：轻量 Markdown 渲染

MarkdownRenderer.java 
if (trimmed.matches("^#{1,6}\\s+.*")) {
    String heading = trimmed.replaceFirst("^#{1,6}\\s+", "");
    result.add(new MarkdownLine(cleanInlineMarkdown(heading), MarkdownLineType.HEADING));
    return;
}
中文解释

这是教学版轻量 renderer，不是完整 CommonMark。

它支持标题、引用、分隔线、代码块、表格、编号列表、bullet 列表和行内 code 清理。

目标是让模型常见 Markdown 输出在终端里更容易读。

源码对照：工具输出折叠

ToolBlock.java 
public void finish(String outputText, boolean success, long elapsedMillis) {
    this.outputText = outputText == null ? "" : outputText;
    this.status = success ? ToolStatus.DONE : ToolStatus.FAILED;
    this.elapsedMillis = elapsedMillis;
    this.collapsed = shouldCollapse(this.outputText);
}
中文解释

工具块开始时只有工具名和参数。

工具完成后补输出、成功失败和耗时。

长输出默认折叠，用户可以用 `ctrl+o` 展开或折叠。

源码对照：可选择的斜杠菜单

AgentTuiWindow.java 
if (handleSlashCommandMenuKey(key)) {
    return;
}

if (userInput.equals("/session") || userInput.startsWith("/session ")) {
    handleSessionCommand(userInput);
}
中文解释

输入 `/` 后会展示当前支持的命令。

`↑/↓` 选择命令，`Tab` 或 `Enter` 选中。

不需要参数的命令会直接执行；需要参数的命令只补全前缀。

`/session list` 只读本地 `workspace/sessions/*.jsonl`。

`/session new [name]` 创建并切换到新会话。

`/session use name` 切换已有会话，不经过 LLM。

源码对照：后台任务只更新状态栏

TuiNotificationSink.java 
public void backgroundTaskCompleted(TaskRun run) {
    window.showBackgroundTaskCompleted(run.taskName());
}
中文解释

这是前面说的策略 B：后台任务完成后，不插入新的对话块。

窗口只更新底部状态栏的 completed / failed 计数。

完整运行记录在 `workspace/tasks/runs.jsonl`，需要时再通过命令或 Agent 查询。

源码对照：emoji 过滤

TerminalTextSanitizer.java 
if (isEmojiLike(codePoint)) {
    continue;
}
result.appendCodePoint(codePoint);
中文解释

部分 emoji 是 supplementary-plane 字符，Java 里要两个 char 表示。

Lanterna 某些绘制路径会把它拆坏，终端里显示成两个问号。

这里仅在显示层过滤 emoji，内部消息和发给 LLM 的内容不变。
