Package · com.aster.core.hook

hook 包：把“流程拦截”从事件广播里拆出来。

事件是事后通知，Hook 是流程节点。长期记忆注入、对话引导适合 `beforeLlmRequest`；工具权限、高危工具审查、HITL 适合 `beforeToolCall`；长期记忆抽取适合 `afterRun`。

四张图看懂 hook 包

架构图

AgentLoop
  -> HookRegistry.apply(BEFORE_LLM_REQUEST)
       -> LongTermMemoryInjectHook
       -> GuidanceHook(后续)
  -> LLM
  -> HookRegistry.decide(BEFORE_TOOL_CALL)
       -> ToolPermissionHook(后续)
       -> HumanApprovalHook(后续)
  -> ToolExecutor
  -> HookRegistry.apply(BEFORE_TOOL_RESULT_APPEND)
       -> ToolResultOffloadHook
  -> HookRegistry.fireQuietly(AFTER_RUN)
       -> MemoryExtractionHook

HookRegistry 是可改写、可阻断、可注册扩展的流程拦截层。

文件结构图

core/hook/
  HookPoint.java                    # 生命周期插槽
  HookHandler.java                  # 单个 Hook 处理器
  HookRegistry.java                 # 注册并触发 Hook
  AgentHookPoints.java              # 当前开放的 Hook 点
  BeforeLlmRequestContext.java      # LLM 前可改写 messages/tools
  BeforeToolCallContext.java        # 工具前可审查 toolCall
  BeforeToolResultAppendContext.java # tool 消息写入前可改写结果
  AfterRunContext.java              # run 完成后触发后台动作
  ToolHookDecision.java             # allow / deny / pause
  ToolHookDecisionType.java         # 决策枚举

hook 包只定义流程协议，具体业务 Hook 放回 memory 或后续 permission 包。

时序图

AgentLoop -> ContextPipeline: build()
AgentLoop -> HookRegistry: BEFORE_LLM_REQUEST
HookRegistry -> MemoryHook: 注入长期记忆
AgentLoop -> LLM: stream()
LLM -> AgentLoop: tool_calls

Hook 发生在动作之前或 run 完成之后，不替代事件总线。

流程图

请求前
  -> Hook 可追加 system/user message

工具前
  -> Hook 可允许或拒绝
  -> 拒绝时写回 role=tool 错误消息

结束后
  -> Hook 提交后台任务
  -> 不阻塞主回答返回

后续 HITL 只需要把 pause 决策接到 TUI 审批流程。

源码对照：HookPoint + HookHandler

HookPoint.java / HookHandler.java 
public record HookPoint<C, R>(String name) {
}

@FunctionalInterface
public interface HookHandler<C, R> {
    R handle(C context) throws IOException;
}
中文解释

`HookPoint` 是主流程的稳定插槽，比如 `before_llm_request`。

`HookHandler` 是挂到插槽上的具体逻辑。

新增 Hook 点时，只新增 HookPoint 和对应 Context，不再改大接口。

新增业务逻辑时，只注册新的 HookHandler。

源码对照：长期记忆已经迁到 Hook

AgentRuntimeFactory.java 
HookRegistry hookRegistry = new HookRegistry();
hookRegistry.register(
        AgentHookPoints.BEFORE_LLM_REQUEST,
        new LongTermMemoryInjectHook(memoryStore, memoryPromptRenderer)
);
hookRegistry.register(
        AgentHookPoints.BEFORE_TOOL_RESULT_APPEND,
        new ToolResultOffloadHook(ToolResultOffloader.defaults(objectMapper, WorkspacePaths.TOOL_RESULTS))
);
hookRegistry.register(
        AgentHookPoints.AFTER_RUN,
        new MemoryExtractionHook(backgroundTaskManager)
);
中文解释

长期记忆注入不再写在 AgentLoop 里。

每轮结束后的记忆抽取也不再伪装成普通事件监听器。

工具结果卸载也不再硬塞在 AgentLoop，统一挂到工具结果写入前 Hook。

它们都是 HookRegistry 中注册的 Hook。
