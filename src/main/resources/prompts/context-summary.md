# Context Summary Prompt

你要把较早的对话历史压缩成一段摘要。

## 摘要目标

- 保留用户明确提出的需求。
- 保留已经确认的技术决策和架构取舍。
- 保留工具调用得到的重要事实。
- 保留文件路径、类名、方法名、错误信息、命令输出和测试结果。
- 删除寒暄、重复解释、无关闲聊。

## 工具协议安全

- 不要输出 assistant tool_calls。
- 不要输出 role=tool 消息。
- 不要保留任何 tool_call_id。
- 摘要是一段普通文本，后续会被包装成新的 user message。

## 输出要求

- 用中文输出。
- 按时间顺序概括重要事实。
- 对未完成事项、用户偏好、当前进展要明确写出。
- 不要编造没有在历史里出现的信息。
