Package · dev.agentmvp.session

session 包：保存事实来源，不做有损压缩。

SessionStore 的职责很窄：保存完整原始消息。真正决定“本轮发给模型什么”的是 ContextBuilder，不是 SessionStore。

两层分工

SessionStore完整原始历史，可用于恢复、调试、重新压缩。
ContextBuilder每次请求前构造实际上下文，可压缩旧 turn。
Long Memory本 MVP 没做，未来可保存偏好、事实、用户资料。

源码对照：内存版 SessionStore

InMemorySessionStore.java 
public class InMemorySessionStore implements SessionStore {
    private final List<Message> messages = new ArrayList<>();

    @Override
    public void append(Message message) {
        messages.add(message);
    }

    @Override
    public List<Message> loadMessages() {
        // SessionStore 保留完整原始对话转写；ContextBuilder 决定本轮实际发什么。
        return List.copyOf(messages);
    }
}
中文解释

`append` 只是追加原始 message，不做摘要、不做截断。

`loadMessages` 返回副本，避免外部直接改内部列表。

这个设计把“存储事实”和“构造上下文”拆开。

为什么这和 PaiCLI 的短期记忆不同？

这里没有单独做“短期记忆层”。当前 MVP 是 session 持久化思路：完整历史先存起来，真正进模型窗口的消息由 ContextBuilder 临时构造。
