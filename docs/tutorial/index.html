Navigation · 当前进展版

从一次用户输入，读懂一个 Java Agent MVP。

这份教程按 Java 顶层包拆分：一个包一个 HTML。每页都包含真实源码片段和中文解释，重点讲清楚当前项目已经完成的 AgentLoop、上下文压缩、MCP、内置工具、Skill 适配层、DeepSeek thinking 和 Lanterna TUI。

说明：`model`、`builtin`、`server` 这类子包会并入所属顶层包页面讲解，避免导航页被纯数据模型页面冲散。

当前进展总览

1

LLM 只走流式。项目通过 OpenAI-compatible SSE 接收模型增量，支持 DeepSeek `reasoning_content` 和流式 usage。

2

工具支持并行。模型一次返回多个 tool_call 时，`ParallelToolExecutor` 会并发执行。

3

内置工具独立成类。`read`、`write`、`bash`、`edit` 都实现 `BuiltinTool`，最后统一注册进 `ToolRegistry`。

4

TUI 是结构化展示层。`UiBlock` 区分用户、thinking、工具和正文，输入框下方展示 token usage 和上下文占比。

按包学习

00

入口包

`TuiMain` 和 `AgentRuntimeFactory` 把供应商、工具、Skill、Session 和 AgentLoop 串成一个可跑的 TUI。

 01

agent

主循环：用户输入、上下文构造、流式模型、工具执行、结果回灌。

 02

context

按 user turn 压缩上下文，避免切断 `tool_call` 和 `tool_result`。

 03

tool

统一工具层：本地工具、MCP 工具和内置工具都变成同一种 Tool。

 04

skill

Skill 渐进式加载：索引常驻，正文按需，references/scripts 复用 read/bash。

 05

llm

DeepSeek 是默认例子，支持 thinking、reasoning_effort、reasoning_content 和 token usage。

 06

mcp

JSON-RPC、tools/list、tools/call，以及通过 workspace/mcp.json 接入本地 stdio/HTTP MCP。

 07

session

JSONL 事件日志保存完整历史，支持恢复、分支和审计。

 08

tui

结构化 UiBlock、Markdown 渲染、底部 usage 预算行、emoji fallback 和工具输出折叠。

 09

chat

OpenAI 兼容消息协议：system、user、assistant、tool 的边界和字段。
