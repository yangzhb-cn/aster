Package · dev.agentmvp.skill

skill 包：只做适配，不做完整 Skill 系统。

这里的 Skill 不是可执行插件。它更像一本按场景摆放的操作手册：启动时只告诉模型“有哪些手册”，需要时再用 `load_skill` 读取全文。

Skill 的脚本和 reference 不自动执行，仍由 read/bash 等工具按需处理。

时序图

Factory -> SkillRepository: scan(workspace/skills)
Factory -> SkillIndexRenderer: render(metadata)
Factory -> SessionStore: append(system skill index)
LLM -> ToolRegistry: load_skill(name)
LoadSkillTool -> SkillRepository: load(name)
LoadSkillTool -> LLM: SKILL.md content

LLM 根据索引自行判断是否需要加载某个 Skill。

流程图

启动扫描 Skill
  -> 只读取 name/description
  -> 注入 system prompt
  -> 用户提出任务
  -> LLM 判断需要哪个 Skill
  -> 调用 load_skill
  -> 读取完整 SKILL.md
  -> 继续使用 read/bash 处理附属资料

这就是渐进式加载：常驻信息尽量小，细节按需拿。

当前实现范围

1

按 `workspace/skills/*/SKILL.md` 扫描本地目录，放进去就能被索引。

2

只解析 frontmatter 里的 `name` 和 `description`。

3

把索引注入 system prompt，让 LLM 自己判断是否需要加载。

4

通过 `load_skill(name)` 渐进式读取完整 `SKILL.md`。

源码对照：扫描 Skill 索引

SkillRepository.java 
public static SkillRepository scan(Path skillsDirectory) throws IOException {
    Path root = skillsDirectory.toAbsolutePath().normalize();
    if (!Files.isDirectory(root)) {
        return new SkillRepository(Map.of());
    }

    List<Path> skillFiles = new ArrayList<>();
    try (var stream = Files.list(root)) {
        stream
                .filter(Files::isDirectory)
                .map(directory -> directory.resolve(SKILL_FILE_NAME))
                .filter(Files::isRegularFile)
                .sorted(Comparator.comparing(path -> path.getParent().getFileName().toString()))
                .forEach(skillFiles::add);
    }
中文解释

目录不存在时返回空仓库，CLI 仍能正常启动。

只看一级子目录下的 `SKILL.md`，所以新增 Skill 不需要改 Java 代码。

这里只收集文件路径，还没有把完整正文塞进上下文。

源码对照：按 name 加载全文

SkillRepository.java 
public SkillDocument load(String name) throws IOException {
    SkillMetadata metadata = skillsByName.get(name);
    if (metadata == null) {
        throw new IllegalArgumentException("未知 Skill: " + name);
    }

    String content = Files.readString(metadata.skillFile(), StandardCharsets.UTF_8);
    return new SkillDocument(metadata.name(), content);
}
中文解释

`load_skill` 不接收路径，只接收扫描过的 Skill name。

这避免把它变成另一个任意文件读取工具。

返回的是完整 `SKILL.md`，但 references/scripts 仍然不会自动加载。
