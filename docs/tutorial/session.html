Package · dev.agentmvp.session

session 包：用 JSONL 事件日志保存事实来源。

SessionStore 不再只是内存 messages。当前运行时把历史写入 `workspace/sessions/default.jsonl`，通过事件回放恢复当前分支的完整消息，再交给 ContextBuilder 构造模型上下文。

三层分工

JsonlSessionStoreappend-only 事件日志，可恢复、可审计。
SessionReplayer按 branch 回放消息，并裁剪半截工具协议。
ContextBuilder每次请求前构造实际上下文，可压缩旧 turn。

源码对照：追加写 JSONL 事件

JsonlSessionStore.java 
@Override
public synchronized void append(Message message) throws IOException {
    appendEvent(SessionEvent.draft(
            sessionId,
            branchId,
            SessionEventType.MESSAGE_APPENDED,
            activeRunId,
            null,
            null,
            message,
            null
    ));
}
中文解释

每次追加 message，都会变成一条 `message_appended` 事件。

事件写入 JSONL 文件，不重写整份历史。

事件里带 `prevHash/hash`，后续可以用于审计。

源码对照：按分支回放

SessionReplayer.java 
if (branch.parentBranchId() != null) {
    result.addAll(selectMessageEvents(
            events,
            branches,
            branch.parentBranchId(),
            branch.forkSeq(),
            visiting
    ));
}
中文解释

分支不复制整份历史，只记录父分支和 forkSeq。

回放子分支时，先拿父分支 forkSeq 之前的消息。

再叠加子分支自己的 `message_appended` 事件。

如何开启新会话

1

输入 `/session list` 查看 `workspace/sessions` 下已有 JSONL 会话。

2

输入 `/session new` 创建自动命名的新会话，或 `/session new demo` 创建指定名称。

3

输入 `/session use demo` 切换回已有会话。

4

切换会话时 TUI 会关闭旧 runtime，重新绑定对应 JSONL 文件。

i
为什么这和 PaiCLI 的短期记忆不同？

这里没有单独做“短期记忆层”。当前 MVP 是 session 事件日志：完整历史先存成可回放事实，真正进模型窗口的消息由 ContextBuilder 临时构造。
