Package · dev.agentmvp

入口包：TUI 负责界面，RuntimeFactory 负责装配。

当前项目不再保留 CLI。`TuiMain` 只启动 Lanterna 终端界面，真正的模型、工具、Skill、Session 和 AgentLoop 都由 `AgentRuntimeFactory` 组装。

源码对照

TuiMain.java 
DefaultTerminalFactory terminalFactory = new DefaultTerminalFactory()
        // 教学版希望跑在当前终端里，而不是弹出 Swing 窗口。
        .setForceTextTerminal(true)
        .setPreferTerminalEmulator(false);
Screen screen = terminalFactory.createScreen();

screen.startScreen();
AgentTuiWindow window = new AgentTuiWindow(screen);
runtime = new AgentRuntimeFactory().create(new TuiAgentEventHandler(window));
window.setRuntime(runtime);
window.run();
中文解释

入口层只创建屏幕和窗口，不直接写 Agent 主循环。

`setForceTextTerminal(true)` 避免 Lanterna 在某些环境里弹出 SwingTerminalFrame。

`AgentRuntimeFactory` 返回完整运行时，TUI 只负责把事件渲染到界面。

这种拆法让 Web、TUI、未来其他入口都能复用同一套 AgentRuntime。

!
关键判断

入口层只负责界面启动和运行时装配，不应该写模型请求、工具执行或上下文压缩逻辑。这样以后要加 Web 入口时，可以继续复用 AgentRuntimeFactory。
